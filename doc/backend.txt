1. Project Structure Patterns:
●	Repository Layer: IGenericRepository<T> -> GenericRepository<T> (Base CRUD).
●	Specific Repositories: I{Entity}Repository : IGenericRepository<{Entity}>.
●	Unit of Work: IUnitOfWork (Quản lý tập trung các Repositories và SaveChangesAsync).
●	Service Layer: I{Entity}Service -> {Entity}Service (Chứa Business Logic, gọi qua IUnitOfWork).
2. Core Generic Implementation:
●	GetAllAsync, GetByIdAsync, AddAsync (có CancellationToken).
●	Update, Delete (Sync).

HUBS
NotificationHub : SignalR Hub dùng để gửi thông báo realtime cho người dùng sau khi đăng nhập bằng JWT. Hub yêu cầu [Authorize] nên chỉ user hợp lệ mới kết nối được.
OnConnectedAsync() : Hàm xử lý khi client kết nối tới SignalR, lấy UserIdentifier từ JWT rồi thêm connection vào group theo userId để server có thể gửi notification đúng người dùng.
OnDisconnectedAsync(Exception? exception) : Hàm xử lý khi client ngắt kết nối, tự động remove connection khỏi group userId để tránh gửi realtime tới connection không còn hoạt động.
Frontend sử dụng:
●	Kết nối tới NotificationHub bằng access token JWT.
●	Lắng nghe event "NewNotification" để nhận thông báo realtime ngay khi backend gửi.
●	Mỗi user sẽ chỉ nhận notification thuộc group của chính mình.
●	Notification realtime thường dùng cho:
○	Xác nhận lịch khám
○	Hủy lịch khám
○	Nhắc lịch khám
○	Thông báo hệ thống
○	Cập nhật trạng thái lịch hẹn
Luồng hoạt động:
●	Backend gọi NotificationService.SendNotificationAsync()
