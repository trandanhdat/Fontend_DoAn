MÔ TẢ FRONTEND – HỆ THỐNG QUẢN LÝ PHÒNG KHÁM
React TypeScript – Tài liệu mô tả dự án đồ án
1. Tổng quan dự án
Hệ thống đặt lịch khám bệnh trực tuyến phục vụ 3 nhóm người dùng: Admin, Bác sĩ (Doctor) và Bệnh nhân (Patient). Frontend được xây dựng bằng React 18 + TypeScript, tích hợp realtime qua SignalR và AI Chatbot Gemini.

1.1 Công nghệ sử dụng
•	React 18 + TypeScript: Nền tảng chính
•	React Router v6: Định tuyến, bảo vệ route theo role
•	Axios: HTTP client, kèm interceptor xử lý JWT/refresh token
•	TanStack Query (React Query): Quản lý server state, caching, loading/error tự động
•	Zustand: Client state: auth, notification
•	Tailwind CSS + shadcn/ui: Styling và component library
•	@microsoft/signalr: Nhận thông báo realtime từ server
•	React Hook Form + Zod: Quản lý và validate form
•	TipTap / React Quill: Rich text editor cho bài viết y khoa
•	Recharts: Biểu đồ cho Admin Dashboard
•	date-fns: Xử lý và format ngày giờ
•	react-hot-toast / sonner: Toast notification thành công / lỗi

2. Cấu trúc dự án
2.1 Thư mục src/
•	assets/: Hình ảnh, icon tĩnh

5. Authentication & Authorization
•	Lưu trữ token: accessToken lưu trong Zustand store (memory), không lưu vào localStorage. refreshToken lưu trong HttpOnly Cookie do backend set
•	Axios Interceptor: Request interceptor đính kèm Authorization: Bearer {token}. Response interceptor khi gặp 401 tự động gọi POST /api/auth/refresh-token, lấy token mới và retry request gốc
•	ProtectedRoute: Component kiểm tra isAuthenticated. Nếu chưa đăng nhập → redirect /login
•	RoleRoute: Component kiểm tra role (Admin/Doctor/Patient). Nếu không đủ quyền → redirect trang 403 hoặc trang chủ
•	AuthContext: Cung cấp { user, isAuthenticated, login, logout } cho toàn bộ app

6. Realtime – SignalR
•	useSignalR hook: Tạo kết nối HubConnection khi user đăng nhập (đính kèm JWT). Lắng nghe sự kiện NewNotification → cập nhật notificationStore. Tự động reconnect khi mất kết nối
•	NotificationBell: Icon chuông ở Navbar, badge hiển thị số thông báo chưa đọc (từ notificationStore)
•	NotificationDropdown: Dropdown top 10 thông báo mới. Click từng item → đánh dấu đã đọc (PATCH /api/notifications/{id}/read)
•	Kích hoạt: SignalR chỉ kết nối sau khi đăng nhập thành công, ngắt kết nối khi logout

7. AI Chatbot – Gemini
•	ChatbotWidget: Nút float tròn ở góc phải màn hình (mọi trang). Click → mở ChatWindow
•	ChatWindow: Cửa sổ chat popup. Lịch sử tin nhắn lưu trong local state (reset khi đóng). Hiển thị typing indicator khi chờ response
•	ChatInput: Textarea gửi tin nhắn khi nhấn Enter hoặc nút Send. Gọi POST /api/chatbot/ask
•	Intent shortcut: Nếu response trả về intent = "book_appointment", hiện nút shortcut "Đặt lịch ngay" dẫn tới BookAppointmentPage
•	Mobile: ChatWindow hiển thị fullscreen trên mobile

9. State Management
9.1 Zustand Stores
•	authStore: { user: UserProfile | null, accessToken: string | null, isAuthenticated: boolean, login(), logout(), setToken() }
•	notificationStore: { notifications: NotificationDto[], unreadCount: number, addNotification(), markAsRead(), fetchAll() }

9.2 TanStack Query
Dùng cho tất cả API calls. Các query key được đặt tên theo module (e.g. ["doctors"], ["appointment", id]). Stale time: 5 phút cho dữ liệu tĩnh (chuyên khoa, bác sĩ), 30 giây cho lịch hẹn/slot.

10. UX / UI & Quy ước
•	Theme: Màu sắc y tế: xanh dương (#2E86AB), trắng, xám nhạt. Font chữ rõ ràng, chuyên nghiệp
•	Responsive: Hỗ trợ mobile (375px), tablet (768px), desktop (1280px+)
•	Loading states: Skeleton loader cho card/danh sách. Spinner cho các action button
•	Toast: react-hot-toast hiện thông báo thành công/lỗi ở góc trên phải
•	Confirm dialog: Modal xác nhận trước khi hủy lịch, xóa dữ liệu
•	Empty state: Hình minh họa + text khi danh sách trống
•	Error boundary: Bắt lỗi runtime toàn cục, hiển thị trang lỗi thân thiện
•	Form validation: Lỗi hiển thị ngay dưới field, không cần submit. Disable nút Submit khi đang loading

11. Checklist Tích hợp Backend
•	Tất cả endpoint backend đều có service tương ứng ở frontend
•	JWT refresh token được xử lý tự động qua Axios interceptor
•	SignalR kết nối sau login, ngắt sau logout
•	Chatbot widget có mặt ở mọi trang (kể cả public)
•	Role-based routing: Admin/Doctor/Patient không truy cập chéo trang của nhau
•	Xử lý lỗi API: 400 (validation), 401 (token hết hạn), 403 (không đủ quyền), 404, 500
•	Appointment status flow: Pending → Confirmed (Doctor) → Completed (Doctor tạo MedicalRecord) | Cancelled
•	Article flow: Draft → Pending (Doctor submit) → Published (Admin duyệt)
•	TimeSlot màu: xanh = Available, đỏ = Booked, xám = Blocked
•	Review chỉ mở sau khi Appointment = Completed và chưa có Review
